## Authentication

+### GitHub OAuth
+
+Sign in at `/api/auth/github`. On callback,
+PushScribe exchanges the OAuth code for a token,
+fetches the user's primary email, and upserts
+the customer row.
+
+A 7-day session cookie is set on success:
+```
+Set-Cookie: session=<uuid>; HttpOnly; SameSite=Lax
+```
+
+**Env vars required:**
+- `GITHUB_CLIENT_ID`
+- `GITHUB_CLIENT_SECRET`

Features

Nothing you don't.

⚡

Webhook-driven

Triggered the instant you push — no polling, no cron lag. Docs are updated before your team even reviews the code.

🎯

Surgical updates

Only changed sections are rewritten. The rest of your docs stay exactly as you wrote them.

CHANGELOG auto-fill

Every run appends an entry to CHANGELOG.md in standard Keep a Changelog format. Releases write themselves.

🔒

Your GitHub token is stored encrypted per-session. Webhook payloads are verified with HMAC-SHA256.

Run history

Every documentation run is logged with its PR link, token usage, and full output — searchable from your dashboard.

🔄

Daily cron pass

Even if you forget to push, a nightly pass keeps your docs fresh. Never ship stale documentation again.
